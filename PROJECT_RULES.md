# TM Insight Development Rules

## Philosophy

TM Insight dibangunkan sebagai platform Political Intelligence yang moden, modular, scalable dan berasaskan data.

Setiap keputusan pembangunan perlu mengutamakan kebolehselenggaraan (maintainability), kebolehskalaan (scalability) dan pengalaman pengguna (user experience).

---

# Git Workflow

- Setiap Sprint mesti mempunyai sekurang-kurangnya satu Git Commit.
- Setiap Milestone mesti di-push ke GitHub.
- Gunakan Conventional Commits.

Contoh:

feat:
fix:
docs:
refactor:
style:
perf:

---

# Coding Standards

- Gunakan React Components.
- Elakkan duplicate code.
- Gunakan TypeScript.
- Naming menggunakan PascalCase untuk Components.
- Naming menggunakan camelCase untuk variables.

---

# UI Standards

- Tailwind CSS
- Responsive Design
- Clean Interface
- Consistent Spacing
- Professional Dashboard

---

# Documentation

Sentiasa kemaskini:

- CHANGELOG.md
- ROADMAP.md

apabila Release selesai.

---

# Development Principle

1. Build Simple
2. Build Clean
3. Build Modular
4. Build Reusable
5. Build Scalable

---

# Golden Rule

Jangan membina sesuatu yang tidak diperlukan hari ini.

Sentiasa bina asas yang kukuh supaya sistem mudah berkembang pada masa hadapan.
